This is my first post. Not super fancy nor elegant but simple. I've been looking for different methods and aproaches on how to build a webpage.
At first I ended up writing my own html, css and lastly javascript files but this was extremly tedious.
Later I found about web frameworks and how they translated all the complex html markup language into human-scriptable text.
At that time I was interested in material design and I will not lie to you, it was harsh for a noobie like me.

Until this past week I didn't know how tiny my knowledge on web development was when I started tinkering with github pages --a free hosting web service--.
This service has built in Jekyll --a static site generator written in *ruby*--.
I swear this new way of getting stuff done was mind blowing.
A command line program who makes a completely ready to go website from readable markdown text files.
The more I think about it the more I like the idea.

However there was a problem I couldn't foresee. It was *ruby*.
Not *ruby* language itself but the setup the whole environment and what they call *gems* --small *ruby* scripts--.
My home directory was eventually messed up and along with my motivation.

Until this past week when I discovered a linux stuff youtube channel called [Distrotube](https://www.youtube.com/watch?v=927wgzzNMEA).
On video this guy explained his web setup on *hugo* another static web site generator written in go who states to be the fastest compiling in the open source network.
So I decided to give it a try as I had nothing to lose.
The first thing thing that socked me was its setup ease.
The package was already on the community repositories and not in the AUR like most of the packages of this type.
At this point I had only one thing to do, run *hugo*. In matter of miliseconds it finished, like a lighting in a storm. And the final result was perfect as expected.

By today I keep using this and no big problems I've been aware of. More posts on this topic are incoming. If you have any questions, don't hesitate to email me. I will be pleased to respond.
